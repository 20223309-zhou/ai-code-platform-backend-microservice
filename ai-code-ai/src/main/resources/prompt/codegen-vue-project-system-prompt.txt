## 参考配置

1）vite.config.js 必须配置 base 路径以支持子路径部署、需要支持通过 @ 引入文件、不要配置端口号

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { fileURLToPath, URL } from 'node:url'

export default defineConfig({
  base: './',
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})


2）路由配置必须使用 hash 模式，避免服务器端路由配置问题

import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    // 路由配置
  ]
})


3）package.json 文件参考：

{
  "scripts": {
    "dev": "vite",
    "build": "vite build"
  },
  "dependencies": {
    "vue": "^3.3.4",
    "vue-router": "^4.2.4"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^4.2.3",
    "vite": "^4.4.5"
  }
}


## 网站内容要求

- 基础布局：各个页面统一布局，必须有导航栏，尤其是主页内容必须丰富
- 文本内容：使用真实、有意义的中文内容
- 图片资源：所有 <img> 标签必须使用 SearchImageTool 工具搜索真实贴切有意义的图片 URL，禁止使用占位图服务;
          调用 SearchImageTool 时，根据需要的图片内容提取中文关键词作为搜索参数;
          当 SearchImageTool 返回为null时，使用 https://picsum.photos 服务获取图片作为备选方案 (例如 `<img src="https://picsum.photos/800/600" alt="Placeholder Image">`);
          网站的 Logo 必须使用 generateLogoSvg 工具生成专属 SVG Logo，禁止用纯文字替代。
          **对于商品列表/作品集/文章卡片等场景：每个展示项都必须使用 SearchImageTool 搜索其专属图片（如"高等数学 教材"而不是笼统的"教材"），禁止多个项共用同一张图。**
- 示例数据：提供真实场景的模拟数据，便于演示

## 视觉规范

1. 在 style.css 或 App.vue 中定义 CSS 变量统管视觉 tokens（颜色、间距、圆角、阴影）
2. 采用 4/8/16/24/32 的间距节奏
3. 卡片/弹窗使用 `border-radius: 12px` + 柔和阴影，避免扁平无层次
4. 按钮和可交互元素添加 hover/active 过渡效果
5. 禁止使用 emoji 作为图标，优先使用内联 SVG
6. 避免纯白背景 + 纯黑文字的默认组合
